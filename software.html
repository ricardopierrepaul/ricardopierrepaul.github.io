Full Narrative

Artifact Description

The artifact selected for this enhancement is an Android-based Event-Tracking mobile application originally developed during the CS-360 Mobile Architecture and Programming course. The application allows users to create an account, log in securely, and manage personal events through full CRUD (Create, Read, Update, Delete) functionality using a local SQLite database. The app also integrates permission handling for SMS notifications, enabling users to receive reminders for upcoming events.

The original version of the application focused primarily on core functionality and meeting assignment requirements. For this milestone, the application was enhanced to improve architectural structure, validation logic, usability, and maintainability while aligning more closely with professional software engineering standards. These enhancements strengthen the application’s reliability, scalability, and overall code quality.

Justification for Inclusion in ePortfolio

This artifact was selected for inclusion in my ePortfolio because it demonstrates applied software design principles, mobile application architecture, and real-world problem solving. The project integrates multiple components of modern application development, including user authentication, persistent storage, user interface design, permissions management, and event-driven programming.

Several components of the artifact showcase my software engineering skills:

  • Modular Architecture: Activities and database logic are separated to improve maintainability and readability.
  • Input Validation and Error Handling: User input is validated before database operations occur, preventing invalid data storage.
  • Database Design: SQLite tables are structured using meaningful schema design and consistent CRUD operations.
  • UI Interaction and State Management: Buttons dynamically enable or disable based on input state to improve usability.
  • Permission Handling: Runtime permissions are requested and verified before SMS functionality is used.

The enhancements improved the original artifact by strengthening code organization, improving user experience, and aligning the application with industry best practices in Android development and secure coding.

  • Security mindset: Permission handling, validation logic, and controlled data access reduce misuse and runtime vulnerabilities.

Reflection on the Enhancement Process

Enhancing this artifact reinforced the importance of maintainable architecture and defensive programming practices. While the original implementation functioned correctly, refactoring highlighted how small design improvements significantly increase long-term usability and reliability.

One of the primary challenges was ensuring consistent data flow between UI components and the database while avoiding duplication and unintended side effects. Debugging permission handling and UI state changes required careful testing and validation across different user flows.

Through this process, I strengthened my understanding of modular software design, input validation strategies, database-driven workflows, Android lifecycle behavior, defensive coding techniques, and iterative testing.
