Full Narrative

Artifact Description

The artifact selected for this enhancement is an Android-based Event-Tracking mobile application originally developed during the CS-360 Mobile Architecture and Programming course. The application allows users to create an account, log in, and manage personal events through full CRUD functionality using a local SQLite database. The app also integrates permission handling for SMS notifications to support reminders for upcoming events.

Justification for Inclusion in the ePortfolio

This artifact was selected because it demonstrates applied software design principles, mobile application architecture, and real-world problem solving. It integrates user authentication, persistent storage, UI design, permissions management, and event-driven programming. The enhancement improved code organization, usability, and maintainability while aligning more closely with professional engineering standards.

Course Outcome Alignment

These enhancements support program outcomes by demonstrating: designing and evaluating computing solutions, applying modern tools and techniques (Android Studio, Java, SQLite), professional communication through clearer structure and documentation, and a security mindset through validation and controlled permission handling.

Reflection on the Enhancement Process

Enhancing this artifact reinforced the importance of maintainable architecture and defensive programming. Refactoring highlighted how design improvements can increase reliability and long-term usability. A key challenge was ensuring consistent data flow between UI components and the database while avoiding duplication and unintended side effects. This process strengthened my understanding of modular design, validation strategies, database-driven workflows, Android lifecycle behavior, and iterative testing.
