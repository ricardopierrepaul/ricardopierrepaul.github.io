Professional Reflection (CS-499)

Growth and Progress

Throughout the Computer Science program, I strengthened my ability to design software systems, implement solutions using appropriate data structures, and build database-driven applications. Earlier in the program, I focused primarily on getting code to work. By the time I completed full-stack and database projects, my mindset shifted toward scalability, maintainability, and security. This capstone required me to present my work professionally and explain not only what I built, but why I built it that way.

Strengths

  • Breaking complex problems into smaller tasks and building iteratively
  • Designing clear program structure and separating responsibilities
  • Applying secure coding concepts like validation, least privilege, and defense-in-depth

Areas for Improvement

  • Increasing automated testing coverage earlier in the development process
  • Improving documentation consistency across projects
  • Strengthening performance profiling and optimization practices

Secure Coding and Professional Practice

Secure coding is a consistent theme in my work. I learned to avoid leaving security until the end by validating inputs, using structured error handling, and testing for vulnerabilities as part of development. I also recognize that professional software development requires ethical responsibility: protecting user data, minimizing risk, and building systems that behave predictably and transparently.

Career Goals

My goal is to pursue a software development role where I can apply these skills in real-world projects. I want to continue growing in full-stack development and secure coding practices while building a portfolio that reflects professional standards.
