Full Narrative

Artifact Overview

The artifact originates from CS-405 (Secure Coding) and consists of a static analysis exercise using QuestionableCode.cpp. The program intentionally contains defects such as infinite recursion, unsafe pointer usage, improper exception handling, and out-of-bounds array access. The artifact was used to compare compiler diagnostics with findings from Cppcheck, highlighting algorithmic and security risks.

Justification for Inclusion

This artifact demonstrates how algorithm design and data structure usage directly impact software correctness and security. Enhancing the artifact required evaluating control flow, recursion depth, and memory access patterns rather than simply correcting syntax errors.

Description of Enhancements

The enhancement process involved interpreting static analysis warnings and mapping them to algorithmic failures such as missing termination conditions and unsafe indexing logic. This strengthened the artifact by demonstrating structured reasoning about execution risks.

Reflection

Enhancing this artifact reinforced the relationship between algorithms, data structures, and secure coding practices. The process improved my ability to analyze control flow and memory access patterns while highlighting how algorithmic defects can lead to vulnerabilities.
