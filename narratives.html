Category Narratives (CS-499)

These narratives explain how my selected artifacts demonstrate my growth across key CS-499 categories. Each narrative includes examples of design decisions, challenges, and improvements made through iteration.

1) Software Design & Engineering

Artifact

  • Travlr Getaways (CS-465) – Full-stack MEAN web application

Narrative

In this artifact, I demonstrate software design and engineering by organizing a full-stack application into layers that separate responsibilities (UI, API, and database). The customer-facing portion uses server-rendered pages, while the admin interface uses an Angular SPA, which required designing reusable components and routing that could scale as features expanded. I implemented RESTful endpoints to support CRUD operations and verified integration through iterative testing and debugging.

2) Algorithms & Data Structures

Artifact

  • Item-Tracking Program (IT-145 / CS coursework) – Frequency tracking + histogram

Narrative

This artifact demonstrates my use of data structures and algorithmic thinking to solve a practical problem. I used an associative structure (map/dictionary) to count item frequencies efficiently, supported searches by key, and generated an output histogram. I focused on correctness, readability, and runtime efficiency by using a single pass through the data input to build counts, then performing constant-time lookups for queries.

Evidence

  • Code excerpt showing the counting structure and search function
  • Sample input/output and histogram output

3) Databases

Artifact

  • Grazioso Salvare (CS-340) – MongoDB CRUD + dashboard filtering

Narrative

This artifact demonstrates database skills by designing CRUD operations that support filtering and data retrieval for a dashboard. I separated database logic into a dedicated data access layer to improve maintainability and testing. I also focused on data integrity by validating inputs and ensuring the dashboard interacted with the database through controlled queries rather than direct manipulation from the UI.

  • Repo link, screenshots of dashboard filters, and CRUD methods
  • Description of database design choices and query patterns
